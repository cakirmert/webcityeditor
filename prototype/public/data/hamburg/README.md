# Hamburg Hosted Data

Place small Hamburg CityJSONSeq demo tiles here so GitHub Pages serves them from
the same origin as the app. This directory is not the city-scale Hamburg data
path.

Current demo file:

```text
prototype/public/data/hamburg/hamburg-center-alkis.city.jsonl
```

It is generated by `npm run data:hamburg-center` from official Hamburg ALKIS
building footprints exposed through the public ArcGIS FeatureServer. It is not
surveyed LoD2: footprints and building attributes are real, while heights and
LoD2-style roof geometry are procedurally derived from storey count and ALKIS
`dachform` where present.

The FileLoader reads `prototype/public/data/manifest.json` and only shows hosted
Hamburg quick samples when the referenced file exists. This avoids CORS problems
and avoids broken demo buttons.

Keep this directory for small browser-safe fixtures only. The committed Hamburg
sample is deliberately limited to 180 buildings so it can load from static
hosting without CORS, memory, or long-download problems.

For the larger prepared Hamburg dataset, use the local strict CityJSONSeq
catalog server instead:

```text
cd prototype
npm run data:hamburg-lod2:serve
```

Then use **Hamburg larger dataset -> Connect catalog** in the loader. That path
loads primitive-valid sequence tiles by viewport and fetches nearby tiles as the
map pans, instead of loading the whole city into the browser at once.
